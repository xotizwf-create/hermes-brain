# Code review

## What to check (in order)
1. **Корректность** — делает ли код то, что заявлено; крайние случаи; ошибки.
2. **Безопасность** — ввод пользователя, авторизация, секреты, инъекции (см. `security.md`).
3. **Переиспользование/упрощение** — нет ли дублирования, можно ли проще/меньше.
4. **Эффективность** — лишние запросы/циклы/аллокации; N+1 к БД.
5. **Стиль** — соответствие окружающему коду (см. `coding-standards.md`).

## Self-review before commit
- `git diff` целиком — глазами, не на автомате.
- Убрать отладочный мусор, закомментированный код, лишние логи.
- Проверить, что не утекли секреты/`.env`.

## Reporting
- Findings — конкретно: файл:строка, в чём проблема, как исправить.
- Не плодить замечания ради количества; приоритет — реальные баги.
