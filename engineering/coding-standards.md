# Coding standards

## General
- Простой, читаемый код. Не усложнять архитектуру без причины.
- Каждый модуль — с понятным единственным назначением.
- Имена говорящие; код читается как окружающий код проекта (стиль, идиомы, плотность комментариев — как в репо).
- Ошибки обрабатывать явно, не глотать исключения.
- Секреты — не в коде. Конфиг через env/secret store.

## Before finishing a task
1. Линтер/форматтер проекта — чисто.
2. Тесты проходят (см. `testing.md`). Если что-то упало — сказать прямо, показать вывод.
3. Быстрая проверка безопасности изменений (см. `security.md`).
4. Обновить документацию/`runbook.md`, если изменилась логика или эксплуатация.
5. Один логический коммит на задачу; осмысленное сообщение.

## Definition of done
- Работает и проверено в реальном запуске, не только «должно работать».
- Не сломаны соседние сценарии (регрессии).
- Изменения отражены в changelog мозга, если затрагивают знания/процедуры.
