# Проверка Магнита: наличие и цена

Использовать, когда Александр просит проверить наличие/цену товара в Магните по адресу доставки.

## Важное правило цены

- Не называть цену из поисковой выдачи/webgate `/v2/goods/search` как финальную без оговорки.
- У Магнита цена в поисковой выдаче web API может отличаться от цены в мобильном приложении/карточке/корзине.
- Если пользователь прислал скрин из приложения — считать его более авторитетным для финальной цены, чем web API.
- В ответе явно указывать источник: «по web API выдачи», «по карточке», «по скрину приложения», «в корзине».
- Для финальной цены по возможности сверять карточку товара `/v2/goods/{productId}/stores/{storeCode}` и/или корзину/скрин. Если доступа только к web API — честно писать, что это не гарантия финальной цены приложения.

## Workflow

1. Геокодировать адрес, если нужны координаты.
2. В Магните найти обслуживающий продуктовый магазин доставки через `shops/v1/shop_by_point`.
3. Искать товар в магазине через `/v2/goods/search` по `storeCode`, `storeType=1`, `catalogType=1`.
4. Проверять упаковку по изображениям товара, если пользователь уточняет цвет/вид упаковки.
5. Для цены дополнительно открыть/запросить карточку товара `/v2/goods/{id}/stores/{storeCode}` с `storetype=1&catalogtype=1`.
6. Если цена в API и в скрине/приложении расходится — признать расхождение и не спорить со скрином.

## Формулировка ответа

- Для наличия: «есть/нет», остаток, можно ли взять нужное количество.
- Для цены: «по источнику X цена Y; финальная в приложении может отличаться, если не сверена корзина».
- Если пользователь показывает финальную цену из приложения — исправить ответ на эту цену и объяснить, что предыдущая цена была из поисковой выдачи/API, а не из финальной карточки/корзины приложения.
