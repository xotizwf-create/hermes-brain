# Простые поставки — импорт прайса и дубли кодов

## Когда применять

Если Александр подозревает, что при загрузке прайса пропадают строки с одинаковым кратким названием/кодом, но разным полным наименованием.

## Устойчивое правило

В прайсе одинаковый краткий код не является дублем сам по себе. Для импорта и проверки уникальности строка считается той же самой только по нормализованной паре:

- краткое наименование / код;
- полное наименование.

Если код совпадает, но полное наименование отличается — это отдельная строка прайса, её нельзя пропускать или перетирать.

## Диагностический сценарий

1. Через MCP прочитать общий гайд `prostavki_operator_guide`, чтобы не расходиться с актуальными правилами коннектора.
2. Проверить живую таблицу `price_list_entries` поиском по коду и по характерным словам полного наименования.
3. Если в базе видна только одна строка при ожидаемых нескольких — смотреть серверный импорт прайса, особенно логику поиска существующей записи перед upsert.
4. Опасный паттерн в коде: сначала искать существующую запись только по краткому коду и затем обновлять её. Это приводит к потере вариантов с тем же кодом и другим полным названием.
5. Правильный паттерн: искать/сравнивать существующую запись по нормализованной паре `short_name/code + full_name`; совпадение только по коду должно создавать/сохранять отдельную строку.

## Проверка после фикса

- Импортировать тестовый прайс, где две строки имеют одинаковый код, но разные полные наименования.
- Убедиться, что обе строки присутствуют в `price_list_entries`.
- Проверить поиск в обе стороны:
  - код → все полные наименования по этому коду;
  - полное наименование → правильный краткий код.
- Если код может быть кириллическим/латинским (`СМ02` vs `SM02`), явно проверить фактический вариант в базе и не подменять символы молча.
