# Входящие договоры: `name` vs `docName` при сверке с прайсом

## Урок из сессии

При ручной обработке входящего договора/контракта со спецификацией нельзя просто копировать договорное наименование в оба поля позиции.

Правильное разделение:

- `docName` / «Наименование для документов» — договорное наименование из спецификации, дословно и полностью: все слова, модели, скобки, пробелы после нормализации переносов, ТУ/РУ/страна и т.п.
- `name` / обычное «Наименование» — краткое наименование/код из загруженного прайса, если найдено уверенное соответствие полной строке спецификации.

## Обязательный порядок

1. Извлечь полное наименование из спецификации договора.
2. Сохранить это полное наименование как `docName` без сокращения.
3. Найти соответствие в загруженном прайсе/каталоге (`price_list_entries`) по уникальным словам, бренду, составу и компонентам.
4. Если соответствие уверенное — в `name` поставить краткий код ровно в написании из прайса (`SM02`, `СМ02`, `P0002` и т.д.).
5. Если соответствия нет или есть несколько разных вариантов — не угадывать и не выбирать молча; оставить `name = docName` либо показать варианты/переспросить, если действие необратимое.

## Пример

Если в договоре длинная строка вида «Набор реагентов ... десяти видов наркотиков ...» и по загруженному прайсу это соответствует `SM02`/`СМ02`, то сохранять так:

- `name`: `SM02` или `СМ02` — строго как в прайсе
- `docName`: полная строка из спецификации договора

## Частые ошибки

- Ошибка: `name` = полная строка из договора, хотя в прайсе есть код.  
  Исправление: найти прайсовый код и поставить его в `name`.
- Ошибка: сократить `docName` до кода.  
  Исправление: `docName` должен остаться полным договорным текстом для документов.
- Ошибка: выбрать один вариант по похожему коду, когда у кода несколько разных полных наименований.  
  Исправление: показать все варианты и отличия, либо оставить полное наименование до уточнения.
