# Коммерческие предложения по коду из прайса

Используй это как дополнение к сверке прайса, когда пользователь просит сформировать КП по короткому коду/артикулу из «Простых поставок» (например, `SM02`/`СМ02`) и отправить его на почту.

## Рабочий порядок

1. Перед созданием или отправкой КП обязательно прочитай MCP prompt `commercial_offer_workflow` через коннектор «Простые поставки». Там зафиксированы текущие правила шаблонов, НДС, наценок и отправки email.
2. Разреши код через актуальный прайс в `price_list_entries`:
   - ищи оба варианта похожих символов: латиница `SM02` и кириллица `СМ02`;
   - если найдено несколько строк по одному короткому коду, используй строку с тем полным наименованием/ценой, которая соответствует запросу или контексту; если контекста нет — покажи варианты или выбери только при очевидном единственном совпадении;
   - не подставляй цену из памяти.
3. Количество бери из запроса пользователя, единицу — из прайса/обычно `шт.`, если prompt workflow не требует иначе.
4. Если пользователь говорит «отправь Александру» без email, для этого проекта допустимо использовать сохранённый рабочий адрес Александра из прошлых операций/почтовой адресной книги; если адрес не найден инструментами — переспроси, не угадывай.
5. Перед отправкой проверь финальные параметры: позиция, количество, цена, НДС, шаблоны/наценки, получатель и формат. Если workflow/tool требует подтверждение — сначала покажи пользователю эти параметры.
6. После отправки сообщи только результат и ключевые параметры, без технических таблиц и без раскрытия почтовых/SMTP-секретов.

## Известная пользовательская настройка

Для медицинских КП Александр предпочитает шаблоны: `КП Столяров М.А.`, `КП фарма`, `КП Интеко`; типичные наценки КП2/КП3 — 1,5% и 3%, НДС — 5%. Всегда сверяй это с текущим `commercial_offer_workflow` и явным запросом пользователя: если пользователь задал другие параметры, приоритет у пользователя.
