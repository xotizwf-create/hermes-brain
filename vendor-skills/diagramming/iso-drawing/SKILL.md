# Skill: iso-drawing — прямоугольная изометрия по ГОСТ

## Инструменты

  `/root/.hermes/agent-knowledge/skills/iso-drawing/scripts/iso_gost.py`).
- **Эталон: `scripts/detail22_demo.py`** — читай ПЕРЕД каждой новой деталью, это

- Рендер только в venv шлюза и только с лимитом памяти (правило preflight, бокс 1 ГБ):
  `systemd-run --scope -q -p MemoryMax=350M /usr/local/lib/hermes-agent/venv/bin/python <скрипт> /root/.hermes/outbox`
- Файлы сохранять ТОЛЬКО в `/root/.hermes/outbox`.

## Теория, зашитая в библиотеку (ГОСТ 2.317 — не переоткрывай)
- Оси: X вправо-вниз 30°, Y влево-вниз 30°, Z вверх. `iso()`:
  `u=(x−y)·cos30°, v=z−(x+y)·sin30°`. Наблюдатель видит верх, перёд (грань −Y), правую (+X).
- **Окружность → эллипс**: при приведённых коэффициентах (=1) большая ось эллипса
  **1,22 d**, малая **0,71 d**. Проверка проекции аналитически: горизонтальная окружность
  даёт `u=1,2247·r·cosφ, v=−0,7071·r·sinφ` — ровно 1,22/0,71. Поэтому **никогда не рисуй
  окружность кругом** — только `circle3()`/готовые методы.
- Линии ГОСТ 2.303: `LW_MAIN` (основная), `LW_THIN` (тонкая ≈s/3), стиль `-.` (осевая).
- **Скрытый контур не рисуем вовсе** (ГОСТ 2.317 разрешает) — так чище. Перекрытие тел
  делает белая заливка: строй снизу вверх, каждое тело само закрывает дальние.

## Готовые методы (используй их, не пиши геометрию руками)
- `Sheet(title, subtitle, date)` — лист А4: `frame()`, `title_block()`, `note()`, `save(pdf, png)`.
- `vcylinder(cx,cy,r,z1,z2,dx,dy, draw_top=, draw_bottom=)` — вертикальный цилиндр с
  силуэтом; для тела, стоящего на другом, ставь `draw_bottom=False`; под пазом-вилкой —
  `draw_top=False, draw_bottom=False` (верх достроит `clevis_y`).
- `clevis_y(cx,cy,r,z1,ztop,zfloor,hw,dx,dy)` — **цилиндр с пазом-вилкой**: паз шириной
  `2·hw` прорезан сверху вдоль оси **Y** → две проушины на +X/−X видны раздельно (это
  ключ к читабельности; паз вдоль X сливал бы их). Рисует обе верхние «луны», стенки и дно.
- `hole_xy(cx,cy,z,r,dx,dy)` — отверстие на горизонтальной грани (эллипс + тень глубины).
- `hole_on_cyl(R,axis,c_along,c_z,r,dx,dy)` — устье бокового отверстия на цилиндре Ø2R
  (истинная кривая на поверхности, `axis='x'|'y'`). НЕ плоский эллипс.
- `axis_line(p3a,p3b,dx,dy)` — осевая штрихпунктирная.
- `vdim(p3_lo,p3_hi, side_x,side_y, label, dx,dy)` — вертикальный размер: выносные от
  точек детали к общей размерной линии в точке `(side_x,side_y)`. **side подбирай так,
  чтобы линия попала В ПОЛЕ листа** (см. ниже).
- `leader(p3,dx,dy, elbow=(ex,ey), text=)` — выноска с полкой (Ø, «N отв.»).

## Порядок работы
1. **Прочитай исходный чертёж и выпиши таблицу элементов с размерами**: тела (Ø и интервал
   высот z), отверстия (Ø, количество, окружность центров), пазы (ширина, ось, глубина).
   Сверь сумму высот с габаритом. Размер не читается — спроси владельца, НЕ выдумывай.
2. **Собери деталь снизу вверх** (painter): основание → отверстия на его грани → ступень →
   стойка/вилка → её паз/отверстия. Вырез рисуй сразу после своего тела.
3. **Размеры — мало и чисто** (главная ошибка прошлых попыток — каша): габаритная высота
   справа, высота основания слева — через `vdim`; диаметры и «N отв.» — `leader` наружу.
4. **Лист**: `frame()`, `title_block()`, примечание про ГОСТ 2.317.
5. **Самопроверка ОБЯЗАТЕЛЬНА и итеративна**: рендери PNG, СКАЧАЙ и ПОСМОТРИ его (base64
   через хелпер → Read), пройди чек-лист, **исправь и перерендери — повторяй, пока не
   станет чисто**. Только потом отправляй PDF из outbox. Не отправляй владельцу «черновик».

## Чек-лист качества (глазами по PNG перед отправкой)
- [ ] Все окружности — эллипсы (1,22/0,71), не круги.
- [ ] Ничего не висит в воздухе: тело начинается на грани предыдущего.
- [ ] Паз-вилка читается как ДВЕ раздельные проушины (паз вдоль Y), а не «коробка».
- [ ] Скрытых линий нет; перекрытия корректны (порядок снизу вверх).
- [ ] **Размерные линии и выноски целиком в поле листа** (между рамкой x∈[20,205]), НЕ
      уходят за край и НЕ пересекают контур/друг друга. Текст горизонтальный.
- [ ] Рамка + основная надпись заполнены (название, дата, масштаб).
- [ ] Файл в `/root/.hermes/outbox`, отправка подтверждена (`"ok":true`).

## Частые ошибки (зафиксированы 11–12.06.2026)
- Рисовать изометрию «своим matplotlib-кодом» с annotate-стрелками поверх фигуры —
  НЕТ, только через `iso_gost.py`.
- Паз вдоль X → проушины сливаются в нечитаемую коробку. Делай паз вдоль **Y**.
- `vdim` с большим `side` → размерная линия уезжает ЗА рамку. Подбирай `side_x≈±43,
  side_y≈∓43` (или считай: `u=(side_x−side_y)·0.866`, держи `dx+u` внутри 20…205).
- Боковое отверстие плоским эллипсом «висит» в стороне — бери `hole_on_cyl` (кривая на теле).
- Отправлять, не посмотрев PNG. Смотри всегда; не идеально — правь и перерендери.
