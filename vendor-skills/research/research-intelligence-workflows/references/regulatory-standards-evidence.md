# Regulatory standards evidence notes

Use this when researching whether a Russian standard/regulation sets a deadline or procedural requirement (e.g. laboratory reports, protocols, environmental testing).

## Pattern learned

For requests like “найди документ, регламентирующий срок оформления протокола химического анализа после исследований”:

1. Search for the governing class standard first, not only the sector example.
   - For accredited testing laboratories, the baseline is **ГОСТ ISO/IEC 17025-2019**, especially section **7.8 “Представление отчетов о результатах”**.
   - Key evidence: results must be reviewed and authorized before release; reports/protocols must be clear, accurate, objective, and include information agreed with the customer and required by the method.
   - Important nuance: this standard regulates review/approval/report contents, but normally does **not** set a universal “within N days after testing” issuance deadline.
2. Then search for form/content rules for the protocol.
   - **ГОСТ Р 58973-2020 “Оценка соответствия. Правила к оформлению протоколов испытаний”** is relevant for protocol/report formatting and required attributes/dates.
   - Treat it as evidence about how protocols must be оформлены, not as proof of a fixed issuance deadline unless the exact text says so.
3. For a sector such as wastewater, distinguish two different deadline classes:
   - **Sample handling / storage / start-of-analysis deadlines** in ГОСТ/ПНД Ф methods for water indicators.
   - **Protocol issuance deadline** after research completion.
   These are often conflated; say explicitly which one the source governs.
4. If no universal statutory deadline is found, conclude narrowly: the deadline is likely set by the contract, customer request/technical assignment, lab quality-management procedure, or a specific administrative regulation for the particular control/expertise context.
5. For disputes, suggest requesting the lab’s internal procedure/contract clause and traceability dates: sampling date, analysis completion date, review/authorization date, protocol issue date.

## Output style

- Lead with the answer: whether a fixed deadline was found.
- List authoritative sources and what each actually proves.
- Avoid overclaiming: “регулирует оформление/утверждение, но не устанавливает единый срок N дней” is often the key finding.
- Provide practical wording for a request/claim if the user needs to use the research operationally.
