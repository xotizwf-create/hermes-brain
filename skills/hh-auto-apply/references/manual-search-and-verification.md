# Ручной поиск и проверка вакансий hh.ru

Использовать, когда владелец просит разовый список вакансий, а не автоматический отклик, либо когда штатный браузерный скрипт нельзя безопасно запускать.

## Безопасная последовательность

1. Сделать preflight памяти до запуска локального Chrome. Учитывать резерв не меньше 512 МБ для сервисов; если после резерва нет бюджета, не поднимать Chrome/Xvfb.
2. Открыть публичный поиск hh.ru через browser-инструмент с параметрами:
   - `area=113` — Россия;
   - `schedule=remote` — удалённая работа;
   - `order_by=publication_time` — сначала свежие;
   - несколько тематических запросов: `внедрение ИИ`, `AI automation`, `ИИ-агенты`, `n8n Make LLM интеграции`.
3. Из поисковой выдачи собрать прямые URL и текст карточек. Не считать метку «Можно удалённо» достаточным доказательством: открыть каждую сильную вакансию.
4. В карточке проверить:
   - она открывается и не архивирована;
   - точную дату публикации (`datePosted` в JSON-LD или строку «Вакансия опубликована»);
   - поле «Формат работы»;
   - вилку и тип суммы (на руки / до вычета);
   - фактические обязанности и требования.
5. Строгая удалёнка означает `Формат работы: удалённо`. Карточки «на месте, удалённо или гибрид» и вакансии, где условия прямо говорят «гибрид», не включать в основной список; можно кратко упомянуть отдельно.
6. Отсеять роли, где AI — только декоративное слово: продажи, маркетинг, контент, 1С, поддержка, обычный backend/Python, чистый ML/DS и управление без личного внедрения.
7. Ранжировать по соответствию профилю, а не только по названию. Самые сильные признаки: анализ бизнес-процессов, прототипирование, Make/n8n, API, CRM, LLM/агенты, измеримый эффект внедрения.
8. Сверить вакансии с журналом откликов, если пользователь спрашивает о новых вариантах. Для списка без просьбы об отклике ничего не отправлять работодателям.

## Формат результата

Для каждой вакансии: название-ссылка, компания, зарплата, дата, подтверждённый формат, 1–2 строки сути и честный нюанс. Сначала лучшие совпадения. В конце отдельно написать, что отклики не отправлялись.

Не обещать «все вакансии», если проверена только часть выдачи. Формулировать: «нашёл и вручную проверил N сильных вариантов»; при необходимости указать, что шумные результаты были отфильтрованы.
