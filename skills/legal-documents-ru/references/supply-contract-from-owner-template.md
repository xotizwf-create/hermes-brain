# Договор поставки из шаблона владельца — рабочий порядок

Проверено 2026-07-22 на договоре для ГБУЗ «Красногорская ЦРБ» (картриджи ЭФГ 63/250 20 мкм,
50 шт × 150 ₽ = 7 500 ₽ вкл. НДС 5 %, поставка 30 календарных дней). Полный цикл от «сделай
договор» до файла в Telegram — около 20 минут, из них большая часть на восстановление задачи.

Это дополнение к `SKILL.md`: там оформление с нуля по ГОСТу, здесь — случай «шаблон есть
в системе», когда владелец ждёт документ 1-в-1 по своей форме.

## 0. Если задачу давали серверному агенту, а он не ответил

Не спрашивай владельца заново — восстанови задачу сам, она вся есть на 217.

```bash
# что просили (история диалогов лежит в sqlite, таблица messages)
python3 -c "
import sqlite3, time
c = sqlite3.connect('file:/root/.hermes/state.db?mode=ro', uri=True)
q = 'select timestamp, session_id, substr(content,1,500) from messages where role=? and timestamp>? order by id desc limit 60'
for t, s, txt in c.execute(q, ('user', time.time() - 8*3600)):
    print(time.strftime('%H:%M', time.localtime(t)), txt.replace(chr(10), ' | ')[:400])
"
```

Оттуда берутся все условия сделки. Дальше — где что лежит:

- **шаблон, который прислал владелец** → `/root/.hermes/cache/documents/` (свежий по mtime);
- **готовые файлы владельцу** → `/root/.hermes/outbox/`;
- **что агент уже успел сделать** → `journalctl -u hermes-gateway --since today | grep -i договор`.

Если незаконченный файл уже есть — **не отправляй его, не проверив по пунктам ниже**. 22.07 в
outbox лежал почти готовый `Договор №22.07.2026.docx` со сверкой «всё True», но предмет в нём
остался от чужой закупки («тест-полоски на холестерин» вместо картриджей ЭФГ): агент подставил
цену и количество в документ другого тендера. Сверка, которая не проверяет наименование товара,
ничего не доказывает.

## 1. Инструменты на 217

- `python-docx` есть **только** в venv агента: `/usr/local/lib/hermes-agent/venv/bin/python`
  (в системном `python3` его нет — не трать время на `pip install`).
- LibreOffice (`soffice`) — конвертация `.doc` → `.docx` и `.docx` → `.pdf`.
- Рабочая папка — `/tmp/<job>/`, финальные файлы — `/root/.hermes/outbox/`.

```bash
mkdir -p /tmp/<job> && cp '/root/.hermes/cache/documents/<шаблон>.doc' /tmp/<job>/source.doc
cd /tmp/<job> && soffice --headless --convert-to docx --outdir /tmp/<job> source.doc
```

## 2. Сначала прочитать структуру, потом править

Выведи ВСЕ абзацы с индексами и ВСЕ таблицы по ячейкам (`repr`, а не `print` — важны табуляции,
неразрывные пробелы и хвостовые пробелы, иначе замены молча не сработают). В типовом шаблоне
владельца: таблица 0 — реквизиты сторон, таблица 1 — спецификация, таблица 2 — подписи;
номер и дата договора дублируются в шапке и в приложении №1.

## 3. Замены — точечные, форматирование сохранять

Менять текст первого `run`, остальные `run` в абзаце очищать: так остаются шрифт, кегль и
выравнивание шаблона. Полный набор полей при смене заказчика:

| Что | Где встречается |
|---|---|
| номер и дата договора | шапка + приложение №1 (обе строки) |
| город | шапка |
| наименование заказчика | преамбула, реквизиты, подписи |
| ФИО главврача | преамбула (род. падеж) + две подписи (инициалы) |
| адрес, телефон, e-mail, ИНН/КПП, л/с, банк | ячейка реквизитов заказчика |
| цена прописью + НДС | п. 2.1 и строка НДС под спецификацией |
| срок поставки | п. 5.1 |
| срок действия | п. 9.1 (в шаблоне он устаревает — проверять год) |
| наименование, ед., кол-во, цена, сумма, ИТОГО | спецификация |

НДС считать самому и записывать готовое число: при цене, включающей НДС 5 %,
`НДС = сумма × 5 / 105` (7 500 → 357,14). Формулы после конвертации не выживают.

Реквизиты нового заказчика брать из его же документа (проект контракта из ЕИС в outbox/кэше),
а не выдумывать.

## 4. Проверка — обязательно двусторонняя

Скрипт-сверка по собранному `.docx` (абзацы + все ячейки таблиц) с двумя списками:

- **должно быть**: номер, дата, город, заказчик, ФИО, адрес, ИНН, л/с, почта, наименование
  товара, количество, цена, сумма, цена прописью, НДС в двух местах, срок поставки, срок действия;
- **не должно быть**: прежний заказчик, прежний главврач, прежний адрес/ИНН/л/с, прежний товар,
  прежние количество/цена/сумма/НДС, прежний срок, прежняя дата.

Обязательно печатать спецификацию строками — глазами видно, что подставилось в ячейки.
Затем `soffice --convert-to pdf`, `pdfinfo` (A4, ожидаемое число страниц).

## 5. Отправка

```bash
python3 /root/.hermes/agent-knowledge/scripts/send_telegram_file.py \
  '/root/.hermes/outbox/<Имя>.docx' --caption '<условия одной строкой>'
python3 /root/.hermes/agent-knowledge/scripts/send_telegram_file.py \
  '/root/.hermes/outbox/<Имя>.pdf' --caption 'Он же в PDF'
```

Каждый файл — отдельным вызовом, «отправлено» говорить только после `OK`.
Имя файла — человеческое: `Договор_<Заказчик>_<товар>_<дата>.docx`.

## Грабли

- Не спрашивать владельца о том, что можно достать с сервера: доступ к 217 есть с ПК владельца
  через `.env` в репозитории мозга (paramiko, пароль не печатать).
- Не отправлять чужой полуфабрикат из outbox без сверки предмета договора.
- Слепая замена даты по всему документу ломает ссылки на законы («ФЗ от «22» июля 2026 г. № 44-ФЗ»
  вместо 05.04.2013) — заменять по конкретным абзацам, а не по всему тексту.
- Опечатки шаблона («заключили настоящий контакт») чинить заодно — владелец шлёт документ
  контрагенту.
