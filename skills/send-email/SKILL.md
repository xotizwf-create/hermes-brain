# Skill: send-email — отправка почты с ящика владельца

## Когда использовать
Владелец просит отправить что-то по почте (текст, подборку, файл) — любому адресату.
Письмо уходит **с его собственного Gmail** (`alexxandr.nikitenko@gmail.com`), получатель видит
обычное письмо от владельца (DKIM/DMARC Google в порядке — не спам).

## Почему именно так (контекст)
- Хостер 217.198.12.236 **блокирует весь исходящий SMTP** (25/465/587 — к любым серверам;
  проверено 2026-06-11). `himalaya message send`, msmtp, swaks — всегда `Connection refused`.
  Локальный файрвол ни при чём — менять его бесполезно.
- Gmail API ходит по HTTPS (443) — не блокируется. Скоуп `gmail.send` добавлен в OAuth-токен
  2026-06-11 (см. `skills/google-account`).

## Как отправить
```bash
python3 /root/.hermes/agent-knowledge/skills/send-email/scripts/gmail_send.py \
  --to "адрес@домен" [--to "ещё@адрес"] \
  --subject "Тема" \
  --body "Текст письма (или --body-file /путь/к/тексту.txt)" \
  [--attach /путь/к/файлу ...] [--html]
```
- Несколько получателей — повторить `--to`. Есть `--cc`/`--bcc`.
- Успех = в stdout `SENT id=<messageId>`. Только после этого можно говорить «отправил».
- Ошибка — честно сказать владельцу и показать суть по-русски (не сырой traceback).

## Правила
1. **Письмо наружу = outward-facing действие**: адресата и суть письма подтверждай с владельцем,
   если он не дал их явно в этом же поручении.
2. Никогда не выдумывай адрес получателя. Нет адреса — спроси.
3. Вложения берёт с любого пути на сервере; размер до ~25 МБ (лимит Gmail).
4. Токен лежит в `/root/.hermes/google_token.json` (compat) и
   `/root/.hermes/secure/google_oauth_token.json` (canonical, 600). Если скрипт ругается на
   `invalid_grant`/scope — токен протух или скоуп потерян: процедура переавторизации в
   `skills/google-account` («Adding write access later»).
