# Skill: iso-drawing — прямоугольная изометрия по ГОСТ

## Инструменты
- Библиотека: `skills/iso-drawing/scripts/iso_gost.py` (на сервере:
  `/root/.hermes/agent-knowledge/skills/iso-drawing/scripts/iso_gost.py`).
- Эталонный пример: `scripts/detail22_demo.py` — **читай его перед каждой новой деталью**,
  это образец структуры скрипта и приёмов.
- Рендер только в venv шлюза и только с лимитом памяти (правило preflight):
  `systemd-run --scope -q -p MemoryMax=350M /usr/local/lib/hermes-agent/venv/bin/python <скрипт> /root/.hermes/outbox`
- Файлы сохранять ТОЛЬКО в `/root/.hermes/outbox` (иначе вложение не уйдёт).

## Что даёт библиотека (всё уже по ГОСТ — не изобретай заново)
- `iso()/project()` — прямоугольная изометрия, приведённые коэффициенты =1 (ГОСТ 2.317):
  X вправо-вниз 30°, Y влево-вниз 30°, Z вверх. Окружности проецируются в эллипсы сами —
  НИКОГДА не рисуй окружность кругом.
- `Sheet` — лист А4: `frame()`, `title_block()` (заполни title/subtitle/date), линии по
  ГОСТ 2.303 (`LW_MAIN`/`LW_THIN`, стили `--` штриховая, `-.` осевая), `save(pdf, png)`.
- `vcylinder(cx, cy, r, z1, z2, dx, dy)` — вертикальный цилиндр с заливкой и силуэтом.
- `slot_x(...)` — открытый паз вдоль X, прорезанный сверху (хорды, стенки, дно).
- `hole_xy(...)` — отверстие на горизонтальной грани; `hole_on_vcyl_y(...)` — устье
  бокового отверстия на цилиндре (истинная кривая, не плоский эллипс!).
- `dim(...)` — линейный размер со стрелками; `leader(...)` — выноска с полкой (Ø, «N отв.»).
- `axis_line(...)` — осевые штрихпунктирные.

## Порядок работы
1. **Прочитай исходный чертёж вслух себе**: выпиши таблицу элементов с размерами
   (основание/ступени/стойки: Ø и интервалы высот z; отверстия: Ø, сколько, на какой
   окружности центров; пазы: ширина, направление, глубина). Сверь сумму высот с габаритом.
   Если размер не читается — спроси владельца, не выдумывай.
2. **Собери деталь снизу вверх** (painter's algorithm — каждое следующее тело перекрывает
   предыдущие): основание → отверстия на его верхней грани → ступень → её пазы → стойка →
   её пазы/отверстия. Вырезы рисуй СРАЗУ после своего тела, до следующего.
3. **Оси и окружности центров** — штрихпунктирные тонкие, поверх (это норма ГОСТ).
4. **Размеры — мало и чисто** (главная ошибка прошлого раза — каша из стрелок):
   габаритная высота справа, высота основания слева, диаметры — выносками с полками,
   отверстия — «N отв. Ø…». Полки НЕ должны пересекать друг друга и контур: после рендера
   проверь и подвинь elbow.
5. **Лист**: `frame()`, `title_block()`, мелкое примечание про ГОСТ 2.317. Деталь по центру
   свободного поля (DX≈105, DY подбери по высоте детали).
6. **Самопроверка ОБЯЗАТЕЛЬНА**: рендери и PNG тоже; перед отправкой посмотри PNG
   (или хотя бы прогони чек-лист ниже). Отправляй PDF (sendDocument из outbox), PNG — как
   быстрый превью по желанию владельца.

## Чек-лист качества (прогнать перед отправкой)
- [ ] Все окружности — эллипсы (по построению через `circle3`), большая ось ⟂ отсутствующей оси.
- [ ] Ничего «не висит в воздухе»: каждое тело начинается на верхней грани предыдущего.
- [ ] Невидимое не нарисовано (перекрытия делает заливка тел — порядок снизу вверх соблюдён).
- [ ] Выноски/размеры не пересекают контур и друг друга; текст горизонтальный.
- [ ] Рамка + основная надпись заполнены (название, дата, масштаб 1:1).
- [ ] Файл в `/root/.hermes/outbox`, отправка подтверждена.
