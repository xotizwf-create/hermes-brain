## 1. Подключение MCP-серверов (скилл `connect-mcp`)

**Идея:** владелец кидает URL MCP-сервера → агент подключает его себе, спрашивает имя, и дальше можно
легко включать/выключать и переключаться.

**Поток «кидаю ссылку»:**
1. `probe` — агент подключается к URL и показывает список инструментов, **ничего не сохраняя**.
2. **Агент обязан остановиться и спросить: «Как назовём этот сервер?»** (жёсткое правило — имя сам не
   придумывает; человеческое название, напр. «Простые поставки», само превращается в безопасный id
   `prostye_postavki`).
3. После ответа — `add <имя>`: запись пишется в `~/.hermes/config.yaml` (секция `mcp_servers`,
   каноническая схема `{url, enabled: true}`).

**Управление:**
- **Список:** «покажи mcp» → `hermes_mcp.py list` (🟢 включён / ⚪ выключен).
- **Переключение = включить/выключить** через родное поле `enabled` (не удаляет!). «Выключи X» —
  пауза, запись и секрет остаются; «включи X» — вернуть. «Удали X» — стереть из конфига совсем.
- **Применение — автоматически.** После любого изменения агент сам обновляет MCP-сессию (патч
  gateway ловит изменение `mcp_servers` и тихо переподключает) — `/reload-mcp` руками вводить не надо.
- **Обновление инструментов — на уровне железа:** systemd-таймер `hermes-mcp-refresh.timer` раз в
  сутки (04:10 UTC) перезапускает gateway и заново собирает инструменты со всех серверов — без трат
  лимита модели.

**Безопасность:** URL часто содержит секрет → живёт только в `config.yaml` (600) на сервере; в выводе
и в реестре мозга он замазан. Перед записью — бэкап, есть откат (`rollback`).

**Где:** скилл `skills/connect-mcp/` (+ `scripts/hermes_mcp.py`, `patches/mcp_autoreload_patch.py`,
`systemd/`); реестр подключённого — `connectors/registry.yaml`; модель — `connectors/mcp-servers.md`.
**Подключено:** `prostye_postavki` (miramed32, 19 инструментов).

## 2. Чтение ссылок, сайтов и Google-документов (скилл `read-links`)

**Идея:** владелец кидает ссылку → агент читает содержимое.

- **Обычные сайты/статьи/Google-доки** → `fetch_url.py "<url>"` (полный текст, без трат лимита). Для
  Google сам превращает share-ссылку в export (Docs→текст, Sheets→CSV со всеми вкладками, `--gid` для
  одной вкладки).
- **JS-страницы / за логином** → нативные `browser_navigate` + `browser_snapshot`.
- **«Найди в интернете…»** → `web_search`, потом прочитать лучший результат через `fetch_url.py`.

**Где:** скилл `skills/read-links/` (`scripts/fetch_url.py`).

## 3. Google-аккаунт агента (скилл `google-account`)

**Решение:** агент работает **под аккаунтом владельца через OAuth, только чтение** — видит все доки
владельца без расшаривания (а не отдельный сервис-аккаунт, которому надо шарить каждый документ).

**Что умеет (read-only):** Календарь, Диск, Документы, Таблицы, Презентации, Gmail (почту пока смотрит
himalaya). Скоупы: `drive.readonly`, `spreadsheets.readonly`, `calendar.readonly`, `gmail.readonly`.

**Как пользоваться:**
- Календарь → `gcal_read.py [--days N]`.
- Документы/Таблицы/файлы → кинуть ссылку → `fetch_url.py` (режим `oauth`, читает и закрытые).

**Как настроили (для будущего):**
1. Google Cloud, проект `gen-lang-client-0802797266` → включили API: Drive, Sheets, Calendar, Gmail.
2. Google Auth Platform → Audience: тип **External**.
3. Credentials → **OAuth client ID → Desktop app** → скачали `client_secret_*.json`.
4. **Вход — только с ПК владельца** (Google режет вход с серверного IP, как было с Codex):
   `google_oauth_login.py <client_secret.json>` → браузер → «unverified app» → Advanced → Go to →
   Allow → получили refresh-токен.
5. Токен залили на сервер: `/root/.hermes/secure/google_oauth_token.json` (600, в git нет; ссылка по
   имени `agent/google/oauth-token`). Нюанс переноса: Git-Bash коверкает `/root/...` →
   `MSYS_NO_PATHCONV=1` или заливка через base64 по SSH.
6. Дальше агент работает headless — токен сам обновляется. **Проверено живьём:** вызов Calendar API
   прошёл.

**⚠ Чтобы доступ не отвалился через ~7 дней:** пока OAuth-приложение в статусе *Testing*, Google гасит
refresh-токен на чувствительных скоупах за неделю. Нужно **опубликовать приложение** (Google Auth
Platform → Audience → **Publish app**, Testing → In production; предупреждение «unverified» для личного
использования — норма) и **разово перелогиниться** после публикации.

**Безопасность/ротация:** только чтение (править/слать/удалять нельзя); токен = чтение всего аккаунта,
лежит только на сервере (600). Отзыв — myaccount.google.com → Безопасность → сторонний доступ →
удалить приложение, затем войти заново. Чтобы добавить запись (создавать события, слать письма) —
добавить write-скоуп, перелогиниться, сделать инструмент; это отдельный шаг доверия.

**Где:** скилл `skills/google-account/` (+ `scripts/gcal_read.py`); коннектор-ссылка
`connectors/google-workspace.md`.
