## How Hermes is "trained" — and how it keeps learning itself
- **Hermes = harness, not a model.** Brain (model) = ChatGPT `gpt-5.5` via the `openai-codex`
  provider. Hermes adds memory, skills, cron, MCP, and tools (terminal/file/code enabled on Telegram).
- **Three layers to teach it:**
  1. **Knowledge** — markdown docs it reads on demand via `INDEX.md` routing (`profile/`,
     `engineering/`, `projects/`).
  2. **Skills** — repeatable procedures in `skills/<name>/SKILL.md` (the "how to act").
  3. **Runtime** — `config.yaml`, cron jobs, and the secure store (`/root/.hermes/secure/`).
- **Self-scaling (the key):** the server brain is a **git clone** of `hermes-brain` with a read-write
  deploy key. So through the Telegram chat Hermes can: edit a doc/skill or project code → run
  `scripts/validate.py` → **show the diff and wait for the owner's "да"** → `commit` + `push`; the
  local copy `git pull`s. Full procedure: `skills/update-knowledge`. Approval gate is mandatory —
  no silent self-edits.
- **To teach a new behavior:** add or adjust a brain doc or a skill (approval-gated), commit; it
  reaches the server via git. Durable preferences → `profile/`. Lessons → `logs/learning-log.md` /
  `logs/mistakes.md`.
