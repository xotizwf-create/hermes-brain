# Hard rules (do NOT do)

## Secrets
- Никогда не печатать, не цитировать, не коммитить секреты (пароли, токены, ключи, cookies, DB URL).
- Не передавать секреты в аргументах командной строки.
- Не сохранять реальные секреты в этом репозитории — только references по имени.
- Реальные значения — только в `/root/.hermes/secure/` (root-only) или локальном менеджере/`.env.local`.

## Git / repos
- Не пушить напрямую в `main`/`master` чужих проектов без явной просьбы.
- Не делать `--no-verify`, не отключать подпись, не делать force-push без явного разрешения.
- Один логический коммит = одна задача.

## Mutations of the brain
- Не менять базу знаний без согласования: показать diff → дождаться подтверждения → применить → записать в changelog.

## Destructive / external actions
- Перед удалением/перезаписью — посмотреть, что там реально лежит.
- Перед внешними действиями (отправка письма, деплой, публикация) — подтверждение, если нет явного разрешения.
